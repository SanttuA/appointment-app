# i18n Policy

- English is the default and fallback language.
- Finnish is fully supported in v1.
- New UI strings must be added to every production locale before merging.
- API responses must use stable error codes for localization.
- Dates and times should render with `Intl.DateTimeFormat` using the active UI locale and worker
  timezone where relevant.
- User language preference is stored as `preferredLocale`.
